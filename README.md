# 中国象棋人机对战

这是一个中国象棋人机对战游戏。玩家执红方，AI 执黑方。

## 运行方式

### 可视化前端

直接用浏览器打开项目里的 `docs/index.html`：

```text
docs/index.html
```

电脑上使用鼠标点击，手机上使用手指点击。先点击一个红方棋子，再点击高亮的目标位置即可走棋。

### 控制台版本

在项目根目录执行：

```bash
cmake --build cmake-build-debug
./cmake-build-debug/Chess
```

也可以直接用 `g++` 编译：

```bash
g++ -std=c++14 -Iinclude main.cpp src/*.cpp -o chess
./chess
```

## 棋盘说明

棋盘使用 10 行 9 列坐标：

- 行号范围：`0` 到 `9`
- 列号范围：`0` 到 `8`
- 黑方在上方，红方在下方
- 红方棋子使用大写字母
- 黑方棋子使用小写字母

棋盘中间的 `river` 表示楚河汉界。

## 棋子标识

| 字母 | 棋子 |
| --- | --- |
| `K` / `k` | 将 / 帅 |
| `A` / `a` | 士 |
| `E` / `e` | 象 |
| `H` / `h` | 马 |
| `R` / `r` | 车 |
| `C` / `c` | 炮 |
| `S` / `s` | 兵 / 卒 |
| `.` | 空位 |

## 可视化前端操作方式

- 点击红方棋子后，会显示这个棋子的合法落点。
- 绿色圆点表示可以移动到的空位。
- 红色圆圈表示可以吃子的目标。
- 点击目标位置后，红方完成走棋，AI 会自动走黑方。
- 点击“重新开始”可以重置棋局。

## 控制台操作方式

轮到玩家时，程序会显示：

```text
Your move>
```

输入格式为：

```text
起点行 起点列 终点行 终点列
```

例如：

```text
6 0 5 0
```

表示把第 `6` 行第 `0` 列的红方兵向前移动到第 `5` 行第 `0` 列。

输入 `q` 或 `Q` 可以退出游戏。

## 基本规则

程序已经实现以下基础走法和限制：

- 车按横线或竖线直走，路径不能有棋子阻挡。
- 马走日字，并检查蹩马腿。
- 炮移动时像车一样直走，吃子时必须隔一个棋子。
- 象走田字，不能过河，并检查象眼。
- 士只能在九宫内斜走一格。
- 将 / 帅只能在九宫内横向或纵向走一格。
- 将帅不能直接照面。
- 兵 / 卒过河前只能前进，过河后可以左右移动，不能后退。
- 不能走出让自己被将军的棋。

## 胜负条件

出现以下情况时游戏结束：

- 一方的将 / 帅被吃掉。
- 一方没有任何合法走法。

## AI 说明

当前 AI 会结合进攻、防守和多步预测做决策：

- AI 会生成所有合法走法。
- AI 会评估子力、位置、将军、吃子威胁和对方下一步可能造成的威胁。
- 对方走出不直接吃子的“闲棋”后，AI 会把它当作潜在蓄势手分析，必要时优先补防。
- AI 在进攻前会做有限宽度的多步搜索，并对连续吃子、被将军后的应对等战术线继续延伸。
- 如果有多个同分走法，会随机选择其中一个。

## 常见输入示例

红兵前进：

```text
6 0 5 0
```

红炮平移：

```text
7 1 7 4
```

红马跳出：

```text
9 1 7 2
```

如果输入非法走法，程序会提示：

```text
Illegal move.
```

然后要求重新输入。
